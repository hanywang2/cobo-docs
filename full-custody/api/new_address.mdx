---
title: Get New Deposit Address
api: "POST /v1/custody/new_address/"
description: Generates a new cryptocurrency address for the specified coin.
---
This endpoint generates a new cryptocurrency address for the specified coin and returns it in a JSON response. The endpoint requires authentication and checks if the user has permission to add an address and if the maximum number of addresses has not been exceeded. The 'coin' parameter specifies the cryptocurrency and 'native_segwit' is a boolean flag indicating whether to generate a native segwit address. The response contains the 'coin' and 'address' fields.

<Note>For BTC address, it supports generating segwit (3 address) and native_ Segwit (bc1 address), see below for details. (*
Cobo supports withdrawal to any BTC address type)</Note>


#### Request
<ParamField body="coin" type="String" required>Coin code</ParamField>
<ParamField body="native_segwit" type="Bool">It is only valid for BTC. If it is empty, it will return the Cobo default address format (segwit 3 address). If it is not empty, it will be native_ Segwit (bc1 address)</ParamField>


#### Response
<ResponseField name="coin" type="String">
 Coin code
</ResponseField>
<ResponseField name="address" type="String">
Deposit address(Seperate Memo by vertical bar)
</ResponseField>

<RequestExample>
```python Python

)
```
